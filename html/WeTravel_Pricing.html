WeTravel

A Messaging and UX Audit

Overview

WeTravel is an online trip planner and payment platform targeted towards semi-professionals, boutique agencies and DMCs (Destination Management Companies).

The Challenge

Our challenge is to create a pricing page that is transparent and easy for users to understand.

My Role

Working on a team with three other UX designers, I was closely involved in the end-to-end process.

Skills

User Research | Wireframing | Card Sorting | Task Flow | Prototyping | Lo/Hi-Fidelity Design | Sketch

Outcome

We were able to reach our goal of getting 80% of users to understand the pricing page.

Pricing Page

Our research process is explained in the landing page study. To view the research process, click here.

For the pricing page, our aim was to get users to easily comprehend WeTrave's complicated pricing structure.

Control

We started with testing the original WeTravel page so that we would have a control to compare our results to.

Results

Only 20% were able to accurately explain and calculate the cost to use WeTravel.

  • Users did not understand the 1% circle callout because it had no label
  • Users did not see or understand the 2.9% processing fee
  • Users did not know the processing fee could be passed on

Cycle 1

Hypothesis

If we create a table to summarize the pricing options, then 80% of users can correctly explain the cost to use WeTravel.

Summary of Changes

Results

Only 20% of users were able to correctly calculate the pricing.

  • Users started to understand the 2.9% fee
  • Still felt deceived with the "...just 1%" wording
  • Table was confusing to some, but users wanted to see an example to explain each scenario
Cycle 1 Prototype

Cycle 2

Hypothesis

If we add a pricing example, then 80% of users can correctly explain the cost to use WeTravel

Summary of Changes

Results

60% of users were able to understand the pricing.

  • Users liked seeing an example
  • But not many people clicked the example
  • Users were trying to add the percentages vertically and were confused when they did not add up
Cycle 2 Prototype

Cycle 3

For this cycle, we decided to take a different approach since the table did not seem to be working. We diverged and created three new prototypes. To validate them, we tested each prototype with fifteen people (5 per prototype).

Option C was the winner!

Hypothesis

If we write out the pricing information, then 80% of users can correctly explain the cost to use WeTravel

Summary of Changes

Results

80% of the users were able to understand the pricing page. Success!

  • Users liked how linear it is
  • Users felt it was very clear and straightforward
  • Users felt like it was too wordy
Cycle 3 Prototype

Cycle 4

Hypothesis

If we emphasize the benefit of passing the processing fee on, then 100% of users can correctly explain the cost

Results

Only 60% of users were able to understand the pricing page. We decided to revert back to cycle 3 for our final design.

  • Users thought they could pass on the 1% fee
  • Users felt that using the phrase "save more" was deceiving
  • Users did not like reading the example like a math problem
Cycle 4 Prototype

Takeaways

We found that explaining the pricing in simple terms was most effective and that users did not view the marketing copy positively. If we had more time, I would have liked to explore creating a visual example to accompany the text.
